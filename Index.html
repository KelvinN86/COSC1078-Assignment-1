Project Idea


Overview

The project that I would like to develop is an easily configurable, customisable and modifiable inventory sorter that can be accessed on both a computer/laptop and a mobile device which are both synced together via cloud system. The purpose this application will serve is to have an easy to use tool where you could easily track the inventory of anything, whether it be personal belongings in a house, groceries, or even the stock of items in a warehouse. The application is able to be synced to multiple devices such as another computer or mobile devices so that changes can be made on the fly, such as checking what supplies are low in your home when you are at a store so you know what to purchase.


Motivation

This project will be interesting and useful, because it can serve as a productivity tool and a quality of life tool, designed so that you could organise your belongings and update the stock level of it easily to know what items in a household is low on at the moment, which may inform you to restock.


Description

Goal:

This product is designed to cleanly keep track of the inventory stock of anything and will be easily configurable, customisable, and modifiable.


Functionality:

The functionality of this product will be the ability to create an inventory or multiple if they choose to, then populate it with a list of items that the user will name, and assign amounts to each of them, the user will also be able to add descriptions to the items such as the current location of the item, ID of item, and etc.


Configurability:

The user is able to configure the list of items any way they want in the form of sorting by a filter, such as sorting items alphabetically, by stock amount from low to high or high to low. The user will also be able to configure the view of the inventory, such as having it in different types of list views, or gridded views.


Customisability, and Modifiability:

The user is able to customise and modify the application to look however they want, examples of such are the ability to add icons to items, assign icons to multiple inventories, change text colour, change background colour/image.


User Interface:

The user interface will be minimal and easy to follow/use, steps will be taken to make sure that the user interface is not cluttered so that it does not affect the user experience negatively. The user interface also comes into consideration when designing the configurability, customisability and modifiability of the product.


User Data and Security:

To use this application users are required to create an account using their email and creating a username and password, the email address will be used for two factor authentication. The email address is the only required personal information the product collects from its users. All the account data will be stored using a cloud service and data stored and received will be required to be encrypted using a form of data encryption. Besides the email address, other data will include the additions and changes the user has made within the product itself, the inventory, the names and descriptions of items, the items itself and images uploaded is also confidential to the user, so all data interacting with this product must also be encrypted.


Usage on Multiple Devices and Syncing:

To allow the product to sync to multiple devices such as another desktop or mobile devices, a cloud service is needed to achieve the result. When designing both the desktop application and mobile application, effort will be made to make them both as similar in design as possible to make the user experience as fluid as possible.


Automation:

The user will be able to optionally set automatic mechanisms in the application such as an automatic notification with or without sound that notifies the user when a set item reaches a set amount.


Tools and Technologies

The equipment required:

Computers to develop the product on, and computers and mobile devices to test the product on when developing it.


The software required:

An integrated development system that supports Python 3, such as PyCham, Visual Studio Code and Atom.

A cloud hosting service for applications to store user data such as Back4App, Heroku, or Firebase.

Github is required for group members to collaborate and work on the project.

A way for group members to meet online such as Microsoft Teams, or Discord A tool used to create wireframe prototypes such as Figma, Adobe XD, or Sketch.

An application designing tool for frameworks such as MIT App Inventor, BuildFire.js, or Framework 7.

Services that provide collaboration and communication such as Microsoft Teams, Discord, and Google Drive


Skills Required

The skills that are required for this project would be the ability to code in the Python 3 coding language, as it will serve to develop the back end of both the desktop and mobile application. Group members will need the ability to create prototype wireframes with tools, the ability to create suitable user interfaces for the application with tools, the ability to host data on cloud services with encryption and the ability to thoroughly test the application for possible fixes.

The skills listed are achievable as these skills are taught in the IT units and can also be self taught.


Outcome

For the outcome to be successful, the prototype, development, and testing phases of the project must be met. If all stages of the project are met, then the outcome of the product will be a working and convenient inventory sorter that is easily configurable, customisable and modifiable to the user’s needs.
